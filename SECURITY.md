# Security Policy

## Reporting a Vulnerability

If you believe you have found a security issue in `pyenv-doctor`, please do not report it publicly in a GitHub issue.

Instead, please use a private security report if that option is available, or contact the maintainer directly.

When reporting a vulnerability, please include:

- a short summary of the issue
- steps to reproduce it
- the potential impact
- any suggested mitigation, if known

## Project Status

`pyenv-doctor` is still an early-stage project.

Security-related reports are welcome, but response times may vary.
